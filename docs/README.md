# Documentación Funcional Base QA

Esta carpeta funciona como referencia funcional para las actividades iniciales de QA.

La documentación principal del proyecto se mantiene centralizada dentro del repositorio general del proyecto para facilitar la alineación interdisciplinaria entre UX/UI, Frontend, Backend, Data y QA.

## Documentación utilizada

- Análisis Funcional Inicial
- Criterios de Aceptación Iniciales
- Transcripción de relevamiento institucional
- Matriz inicial de Test Cases

## Objetivo

La documentación mencionada sirve como base para:

- definición de escenarios de prueba
- diseño de test cases
- construcción de automatizaciones futuras
- trazabilidad funcional
- validaciones QA

## Observación

Los documentos continúan siendo evolutivos y podrán refinarse a medida que el cliente proporcione nueva información institucional.
